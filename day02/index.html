提问引入

从一个简单的问题开始

Question 01

❓ OpenClaw 和豆包、DeepSeek 这些对话框有什么区别？

💡 答案

不是豆包笨——是没人告诉它。

OpenClaw 之所以懂穹狼，是因为有人给它写了一份"手册"。

Key Difference

🔑 最根本的区别是什么？

❌ 豆包 / DeepSeek

  • 没有记忆机制
  • 没法设定系统提示词
  • 每次打开不知道你是谁
  • 上次说的话全忘了

✅ OpenClaw / Claude Code

  • 有记忆机制和规则机制
  • 可设定系统提示词
  • 写规则文件自动加载
  • 越用越懂你

🦞

养 龙 虾

它不是买回来就能用的成品

你得喂它规则、调它的配置、让它积累记忆
它才会越来越懂你

上下文窗口

AI 的"脑子"到底是怎么运作的？

Concept

上下文窗口 = AI 的"工作台"

它能看到的全部信息，都放在这张"工作台"上

📥

有限容量

200K

tokens ≈ 15万字

📋

实时拼装

每次对话都重新
组装内容

🗑️

会溢出

满了就清掉
早期内容

Boot Sequence

上下文窗口是怎么"开机"的

1
你发了一条消息
消息进入窗口，触发组装流程
2
System Prompt 加载
Anthropic 的出厂指令，看不到也改不了
3
全局 CLAUDE.md 加载
你自己写的个人规范
4
项目 CLAUDE.md 加载
公司级别的规则——穹狼的信息都在这
5
Rules 文件加载
岗位级别的细则——反馈规范、话术规范等
6
Memory 加载 → 开机完成
所有手册拼好了，Claude 准备就绪

⚠️ Alert

窗口满了！

100% — 已满

被清掉的 ❌

  • 早期的对话记录
  • 之前读取的数据
  • 旧的工具调用结果
  • 口头说过的偏好

留下来的 🔒

  • CLAUDE.md（常驻）
  • Rules 文件（常驻）
  • Memory（常驻）
  • System Prompt（常驻）

💡 口头说的会被清掉，写进手册的永远在

03

陷阱问答

检验你是否真正理解了

Trap #1

🤔 你跟 Claude 聊了 2 小时，它是不是越来越懂你了？

⚡ 陷阱

大部分人会说"是"

✅ 答案

不是！早期对话已被卸载

聊了 2 小时窗口快满了，早期的对话反而被卸载了——它不是越来越懂你，是越来越只记得最近说的话。想让它一直懂你，要写进 CLAUDE.md。

Trap #2

🤔 我在 CLAUDE.md 里写了"课后反馈要用温暖语气"，第二天开新对话，Claude 还记得吗？

⚡ 陷阱

有人会说"不记得"

✅ 答案

记得！CLAUDE.md 是常驻层

CLAUDE.md 是常驻层，每次新对话都会重新加载。这就是它和"聊天时口头说一句"的本质区别。

Trap #3

🤔 口头说"表格都用蓝色标题"跟写进 CLAUDE.md，效果一样吗？

这一轮一样，下一轮就不一样了！

口头说的只在当前对话有效，窗口满了还可能被卸载。写进 CLAUDE.md 的每次对话都在——这就是"短期记忆"和"长期记忆"的区别。

💎 加分题

让 Claude 查了大众点评的评价，又让它发到钉钉群。哪个更占窗口？

答案：查大众点评远比发钉钉占空间。不是操作复杂就占空间——是返回的数据量大才占空间。

四层记忆体系

从个人到团队，记忆分层管理

Memory Architecture

四层记忆体系

🧠
Auto-memory MEMORY.md

Claude 自动积累 · ≈ 同事久了知道你的习惯

跟着个人走
🪪
全局 CLAUDE.md ~/.claude/CLAUDE.md

你自己写 · ≈ 你的工牌

跟着个人走
📖
项目 CLAUDE.md 项目根目录

团队一起写 · ≈ 穹狼员工手册

跟着项目走
📋
.claude/rules/ .claude/rules/*.md

各岗位写各自的 · ≈ 岗位操作手册

跟着项目走

前两层跟着你个人走（换项目还在），后两层跟着项目走（换人也在）

Difference

CLAUDE.md 是"我是谁"
Rules 是"做这件事时怎么做"

管理层 Rules

写周报时的格式规范
数据对比要带同比

销售 Rules

回复家长咨询的话术
不承诺确定性结果

老师 Rules

课后反馈的模板格式
评估维度和用语规范

CLAUDE.md 一份，rules 可以有很多份

五个管理命令

Commands

五个核心命令

06

记忆积累三种方式

发现新规则，怎么加进去？

Methods

三种方式积累记忆

方式一

# 手动追加

输入 # 把规则追加到 CLAUDE.md

≈ 你自己拿笔补一条

方式二

口头指令更新

直接说"帮我加一条规则"

≈ 口头跟助理说

方式三

Auto-memory

Claude 观察你的习惯，主动提议记住

≈ Claude 自己学

关键区分："你喜欢表格汇报" → 个人习惯 → Auto-memory  ≠  "不提学费" → 公司制度 → CLAUDE.md

🦞

养龙虾

你喂它什么规则，它就按什么规则来

全局 CLAUDE.md → 你是谁（个人工牌）

项目 CLAUDE.md → 穹狼怎么干（员工手册）

.claude/rules/ → 你的岗位怎么干（岗位手册）

Auto-memory → Claude 自己记住的习惯
