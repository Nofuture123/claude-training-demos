核心结论

1
你看到的 ≠ AI 脑子里的全部
右栏只有对话，但中间的上下文窗口里还有 System Prompt、CLAUDE.md、工具定义等大量"隐藏信息"。
2
三种加载时机决定了"谁在脑子里"
🔒 开机就加载（CLAUDE.md、Rules、Memory、MCP 全部、Skills 说明）→ 一直在
⚡ 触发时加载（Skills 完整内容）→ 触发才来
📦 调用时加载（工具执行输出、文件内容）→ 取到才来
3
MCP 开机全量加载，Skills 是唯一的"双阶段"
MCP 开机就把所有工具的完整定义装进上下文，没有第二阶段。
只有 Skills 分两步：名字+简介开机就在，完整指令触发后才来。
4
API 走另一条路——写代码 → 执行在外面 → 结果回来
AI 写代码在上下文内，代码执行和 API 调用在上下文外面，只有执行结果回到上下文。
5
压缩时常驻层会被清理，但会重新注入
上下文压缩时整个窗口都会被清空。但常驻层（CLAUDE.md、Rules 等）会被重新加载回来——不是"没被删"，是"删了又回来"。这就是为什么写好 CLAUDE.md 这么重要。
6
大文件是"容量杀手"，/compact 是急救药
一个 35KB 的文件就能吃掉大量窗口空间。窗口快满时用 /compact 压缩。
